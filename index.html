Formo

Formo is a form module for Kohana 3 that lets you work with forms as objects.

  • Installation
  • Upgrading

Installation

Since Formo was written specifically for Kohana, installation is a breeze. To install Formo in your application: 1. Create a directory for Formo in your modules directory 2. Clone Formo (git://github.com/bmidget/kohana-formo.git) or copy Formo into your new directory 3. Add the Formo module to the modules located in /application/bootstrap.php That’s it, you’re ready to use Formo.

Upgrading

Here are some changes that you need to know in order to upgrade from the last version of Formo - Rules are no longer added using `Formo::rule()` or `Formo::rules()`. Instead you must use `Formo::add_rule()`. [See validation documentation](validation.html) - There is no longer a `Formo::field()` method. Just use `Formo::form()` and `Formo::factory()` - `create_sub()` is now `subform()`, and it returns the form object, not the subform - Formo doesn't do automatic belongs to, has many, or has many through relationships because it's ridiculous to auto-generate these - File fields have a `NULL` empty value, not an empty `$_FILES` array - Out-of-the box styling works now with Twitter Bootstrap. If you need backwards compatibility with the old version of Formo, ste the `template_dir` variable in Formo's config file to `'formo/'`
